Style Guide

These standards keep the docs consistent, readable, and highly practical.

Writing Tone

Practical and direct. Say what to do and why. Avoid “just” and “simply”. Define acronyms on first use. Short paragraphs.

Commands

Must be copy-pasteable. Include exact syntax. Explain any placeholders immediately.

Verification & Tested On

Every guide ends with a verification section. Include a “Tested on” block with date, hardware, and software versions.

DANGER: Use this callout for actions that can cause hardware damage or data loss.

Additional Rules

Use proper callouts, fenced code blocks with titles, consistent headings, and descriptive image filenames. Mark missing info with TODO markers.
