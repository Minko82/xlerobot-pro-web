3-Bus Architecture

Build & Calibration Guide

Consolidated instructions for print, assembly, wiring, and physical validation for the dual-arm XLeRobot platform.

CAUTION: This page documents a 3-bus draft architecture. Some older pages still describe a shared/2-bus layout. For this build variant, follow this page exclusively.

1. Build Overview

This guide walks you through building an XLeRobot platform with 2x SO-101 follower arms, a neck/head assembly, omni-wheel base, and an NVIDIA Jetson compute node.

The 3-Bus Separation

We use three independent motor buses to simplify wiring, isolate power faults, and make debugging significantly easier.

BusSubsystem
Bus 1Left arm
Bus 2Right arm
Bus 3Neck and wheel base

2. Print Settings

Start from your baseline PLA profile. Parts were designed for and tested on consumer printers (e.g., Bambu A1, Prusa XL). Download the pre-configured XLeRobot Print Profiles (.3mf) ↗ to get started immediately.

Neck & Wheel Base

Structural parts requiring high rigidity.

  • Perimeters: 4
  • Solid Layers: 5-6 top/bottom
  • Infill: 25-35% Gyroid (35% for motor mounts/neck)
  • Layer Height: 0.20 mm

Arm Joints (SO-101)

High dimensional accuracy required.

  • Perimeters: 4
  • Solid Layers: 5 top/bottom
  • Infill: 20-30% Gyroid (30% for joint housings)
  • Layer Height: 0.16-0.20 mm
NOTE: The wheel-base connector may need Z scaling only in your slicer to properly balance the load across the omni-wheels and cart wheels.

3. Assembly

A) Cart & Wheel Base

Start by building the rolling structure using the IKEA RÅSKOG Assembly Manual ↗, then proceed directly to the XLeRobot Wheel-Base Integration Steps ↗.

  • Keep omni-wheel orientation exactly as documented.
  • Do not finalize cable routing through the cart yet. Leave access for motor extensions and USB data.

B) Arms (Joints 1-5) & Neck

Complete the SO-101 Follower Arm Assembly ↗ for both the left and right arms. Crucial step: Before attaching the gripper to joint 5, install the wrist camera attachment to avoid having to reopen the wrist stack later.

4. Motor Configuration & IDs

Configure motors before final cable management and hard mounting. Set IDs and baudrates using Bambot (GUI) ↗ or the LeRobot CLI Configurator ↗. Many motors ship with default ID 1. ID assignment must be unique per bus.

BusRecommended IDsMapping Notes
Bus 1 (Left Arm)1-6SO-101 follower mapping
Bus 2 (Right Arm)1-6Valid because bus is separated
Bus 3 (Neck & Wheels)7-117-8 Neck, 9-11 Wheels
Identify Motor Ports
# Run this to identify your active serial ports before assigning IDs
$ python lerobot/find_port.py

# On Linux, you may need to grant serial permissions:
$ sudo chmod 666 /dev/ttyACM0
TIP: Label each motor with a marker after configuration (e.g. L-1, R-1, B-7). When wiring, you need to know exactly which motor goes to which control board.

5. Jetson Placement & Wiring

Mount the Jetson at the base of the neck or in the center storage shell area to keep mass low for stability and minimize cable length to the controller boards.

Power & Camera Routing

  • Power: Mount the battery upright. Use separate fused outputs for the left arm, right arm, neck/wheel bus, and Jetson power path. Add strain relief.
  • Wrist Cameras: Route each cable along its arm with full-motion slack. Secure at intervals (Velcro/clips).
  • Head Camera: Route through the hollow neck, leaving pan/tilt slack at the head joint.

6. Hardware Calibration & Validation

Before moving on to the software stack, you must validate your physical assembly. This ensures that motor IDs are correct, wiring is secure, and there are no mechanical defects affecting the Inverse Kinematics (IK) solver.

A) Joint Control Sanity Check

Test all joints to ensure the buses are wired properly and no motors are skipped or reversed.

Test Joint Movements
$ python 0_so100_keyboard_joint_control.py

B) Mechanical IK Calibration

The analytical IK solver assumes standard SO-101 geometry. Command the arm to a known position and visually confirm the gripper reaches the expected location. If consistently off, check your physical build for loose servo horns or bent brackets. There is currently no software calibration offset to fix bad mechanical assembly.

Verify IK Accuracy
$ python 1_so100_keyboard_ee_control.py

C) USB Camera Connection Check

Verify your physical camera wiring is recognized by the OS (typically even-numbered devices like /dev/video0, /dev/video2).

Test Camera Feeds
# List video devices
$ ls /dev/video*

# Quick camera feed test
$ python -c "import cv2; cap = cv2.VideoCapture(0); ret, frame = cap.read(); print('OK' if ret else 'FAIL'); cap.release()"
